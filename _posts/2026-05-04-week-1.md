Sunday is a shell on top of macOS. The way Aqua is a shell, the way Windows Explorer is a shell — except Sunday's interface is not windows and icons. It is intent. You speak or type what you need, and the screen shows you that it happened. That sentence is the entire product. Everything that follows is the work of making it true on a real machine.

The first journal entry should be the easy one to write, but it is the hardest, because the most honest thing to say is that we threw out the previous prototype. The Tauri and Rust version worked as an app. It did not work as a shell. Tauri creates a window. A shell needs to be more fundamental than a window — it owns the display, hides the Dock, replaces the menu bar, and recedes when an app needs the screen. That is a job for the platform's own toolkit, not a cross-platform layer. Native Swift and SwiftUI was the right answer and we should have arrived there sooner.

## the stack

The architecture is now narrow on purpose. Swift and SwiftUI for the shell. SwiftData with `sqlite-vec` for everything we remember — Cards, the VFS, the environment timeline, vector search. SFSpeechRecognizer for voice input, configured with `requiresOnDeviceRecognition = true` so nothing leaves the machine. AVSpeechSynthesizer for voice output. EventKit, Contacts, PhotoKit, HealthKit, MusicKit, CoreSpotlight, Vision, FSEvents, NSWorkspace, AXUIElement — all the personal-data and orchestration capability is already in the operating system. We just need to use it correctly.

Intelligence is split in two and the split is real, not cosmetic. The System Brain is Apple's Foundation Models framework, a roughly 3B model running on the Neural Engine. It handles classification, structured output, and `RenderSpec` generation through constrained `@Generable` decoding — typed Swift structs the model populates field by field, no JSON parsing, no hallucinated keys. Sub-second. The Deep Brain is a local LLM served by Ollama, used for open-ended reasoning and multi-step planning. Two to five seconds. The Active state on the canvas breathes at 60% speed when the Deep Brain is engaged. Pacing is the only signal of which brain is at work.

## what works

The launch dissolve runs. The desktop fades out as Sunday fades in over 1.8 seconds simultaneously, the time display fades in during the final 0.8, and the shell acquisition — hiding the Dock and menu bar — fires at the end, after the fade has already covered them. Sunday is not a window that appears on top of the desktop. The desktop becomes Sunday.

The three canvas states are wired. Ambient is a black surface with the time in 72pt Thin and the last intent dimmed below it. Active is a single breathing white element, scale 0.7 to 1.0, opacity 0.4 to 1.0, easeInOut, 2.4 second cycle. Surface is the answer. The transition from Active to Surface is the arrival — there is no separate "here is your answer" beat, because that beat is for products that do not trust their own typography.

Voice and text are now equivalent inputs in the literal architectural sense. Both produce the same `Intent` value with a different `source` field. ⌘L starts listening, partial transcripts populate the same `inputBuffer` that typed characters fill, and a 1.2-second silence submits — same submit path either way. When a voice intent surfaces, AVSpeechSynthesizer reads only the headline back. Detail is for the screen, not the speaker. Default voice, default rate, no SSML. Consistency is the trust mechanism.

There is also a `VoicePickCapability`. Saying "use Evan's voice" or "switch to Joelle" matches against the installed Apple voices, persists the choice in `UserDefaults`, and confirms with a brief surface. We do this rather than follow the macOS System Voice setting because there is no public API to read it — third-party apps get the full list and choose for themselves, so the only path that respects the user's preference is to let them tell Sunday directly. That is the principle in action: every preference is a sentence, not a settings panel.

## the brand mark

The site you are reading uses `Sunday...` as its logotype. The ellipsis is rendered in `#6e6e73` at weight 300, against the same true black the canvas uses. It is the wake word pause — the half-second between calling for the system and the system being there. The visual language of the journal is the visual language of the product, by intent. Pure black, white text, SF Pro, no shadows, no gradients, no rounded corners on layout. Once a visual decision is made correctly it is made permanently, so the journal does not get its own design system.

## next

The next slice is the always-on local wake word, which replaces ⌘L as the primary trigger. After that is the days-between tool — a small, deterministic helper that the System Brain can call rather than guessing date arithmetic from training. Past that, the proactive insight surface, which is described in the spec but has not been built. None of this is aspirational. It is what the open plan files in `plans/` say is next, and the next entry in this journal will say what actually happened.
